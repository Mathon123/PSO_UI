# 常见问题 (FAQ)

本文档收集了使用PSO数据分析工具时可能遇到的常见问题及解决方案。

---

## 数据导入问题

### Q1: 提示"无法识别数据格式"

**症状**：导入CSV文件时提示数据格式错误

**原因**：
- 列名不符合规范
- CSV编码问题
- 数据为空或格式错误

**解决方案**：
1. 确保列名包含 `I_peak(mA)` 或类似电流列
2. 确保有频率响应列（如 `Freq_10Hz`）
3. 检查CSV文件编码（推荐UTF-8）

---

### Q2: 数据点数量不足

**症状**：导入后数据显示不完整

**原因**：数据点少于系统最低要求

**解决方案**：
1. 每个频率至少需要3个数据点
2. 建议每个频率有5-10个数据点
3. 电流范围应覆盖阈值附近区域

---

## 优化问题

### Q3: 优化长时间不收敛

**症状**：进度条长时间不动或进度缓慢

**原因分析**：

| 原因 | 可能性 | 表现 |
|------|--------|------|
| 目标RMSE过低 | 高 | 迭代次数达到上限仍未达标 |
| 粒子数不足 | 中 | 搜索能力不足 |
| 数据噪声大 | 低 | RMSE波动剧烈 |

**解决方案**：

```python
# 方案1：放宽目标RMSE
目标RMSE: 0.03 → 0.05

# 方案2：增加粒子数和迭代
粒子数: 50 → 100
迭代次数: 50 → 100

# 方案3：调整惯性权重
w_max: 0.9 → 0.8  (增强局部搜索)
```

---

### Q4: R²为负值

**症状**：R²指标显示为负数

**原因**：模型拟合效果比简单均值预测更差

**正常范围**：
- R² > 0.9：优秀拟合
- R² > 0.7：良好拟合
- R² > 0.5：可接受拟合
- R² < 0：模型不适用

**解决方案**：
1. 增加迭代次数
2. 使用更大的粒子群
3. 检查数据是否符合电刺激响应特征
4. 考虑数据预处理（异常值移除）

---

### Q5: 优化结果不稳定

**症状**：多次运行优化结果差异大

**原因**：算法随机性 + 局部最优

**解决方案**：
1. 增加粒子数
2. 多次运行取最优结果
3. 调整变异概率 `mutation_prob`

---

### Q6: 内存不足错误

**症状**：运行时报内存溢出

**原因**：
- 粒子数设置过大
- 数据量过大
- 同时运行多个优化

**解决方案**：
```python
# 减小粒子数
粒子数: 200 → 50

# 减少数据点
每个频率: 50个点 → 20个点
```

## 图表问题

### Q7: 图表显示空白

**症状**：图表区域没有内容

**原因**：
- 尚未运行优化
- 数据导入失败
- 图表未更新

**解决方案**：
1. 先导入数据
2. 运行PSO优化
3. 查看状态栏提示

---

### Q8: 图表中文显示乱码

**症状**：图表中的中文显示为方块

**解决方案**：

1. 设置中文字体（Windows）：

```python
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['Microsoft YaHei', 'SimHei']
plt.rcParams['axes.unicode_minus'] = False
```

2. 或在配置文件中设置

---

### Q9: 图表保存失败

**症状**：点击保存按钮无响应

**原因**：
- 文件路径包含特殊字符
- 目录权限不足
- 文件被其他程序占用

**解决方案**：
1. 使用简单路径（桌面或文档文件夹）
2. 确保文件名不包含特殊字符
3. 关闭可能占用文件的程序

---

## 模型参数问题

### Q10: 参数辨识结果不合理

**症状**：某些参数值为极端值或NaN

**原因**：优化陷入无效区域

**解决方案**：
1. 检查参数边界设置
2. 验证实验数据范围
3. 查看收敛曲线判断优化质量

---

### Q11: 阈值电流计算失败

**症状**：P50/P80/P90显示为空

**原因**：
- 模型参数无效
- 概率计算超出范围

**解决方案**：
1. 检查优化是否成功
2. 验证RMSE是否达标
3. 重新运行优化

---

## 性能问题

### Q12: 程序运行缓慢

**症状**：界面响应迟钝

**原因**：
- 数据量过大
- 优化参数过大
- 后台进程占用资源

**解决方案**：
1. 减小数据量
2. 降低PSO参数
3. 关闭其他占用资源的程序

---

### Q13: 启动时间过长

**症状**：程序启动需要很长时间

**可能原因**：
- matplotlib字体加载
- 依赖库初始化
- 系统环境问题

**解决方案**：
1. 首次启动后字体会被缓存
2. 确保Python环境正常
3. 考虑使用打包后的可执行文件

---
